# Industra Plan 拆分总览

本目录基于 `project.md` 的总体愿景，将项目按“可独立演进、可并行开发、接口清晰”的方式拆分为若干子产品文档。

## 拆分原则

1. 先按产品能力域拆分，而不是按前后端技术栈拆分。
2. 每个子产品都有明确输入、输出、边界和验收口径。
3. 公共能力下沉为底座，避免多个子产品重复建设。
4. 优先支持 MVP 主链路：`产业 -> 地图 -> 企业 -> 上下游 -> 事件/AI解释`。

## 推荐开发顺序

1. 数据底座与采集平台
2. 产业知识图谱与关系引擎
3. 地理地图与区域产业视图
4. 企业情报与产业链探索台
5. 事件脉冲与产业传导系统
6. AI 产业分析与知识构建助手
7. 平台接口与版本规划持续收口

## 依赖关系

- `02 数据底座` 是所有子产品的上游基础。
- `03 图谱引擎` 依赖 `02`，并向 `04/05/06/07` 提供统一实体与关系查询。
- `04 地图视图` 依赖 `02/03` 的地域和企业数据。
- `05 企业探索台` 依赖 `03/04` 的关系与空间上下文。
- `06 事件系统` 依赖 `02/03` 的实体标准，并向 `07` 输出事件图谱。
- `07 AI 助手` 依赖 `03/06` 的结构化知识和事件上下文。

## 文档使用方式

- 讨论产品方向时，先看 `01` 和 `08`。
- 明确数据与模型时，先看 `02` 和 `03`。
- 推进前台产品开发时，重点看 `04`、`05`、`06`、`07`。
- 每个子产品都可单独继续细化为 PRD、接口文档、任务拆分和验收清单。
