# 1. 项目简介

Industra 是一个：

# 「产业地图 + 知识图谱 + 地理可视化」

系统。

它将：

- 股票市场
- 产业链
- 公司关系
- 区域经济
- 实时新闻
- AI 分析
- 动态事件

融合为一个：

# 可探索、可推演、可视化的产业宇宙

用户能够像浏览 Google Maps 一样探索产业世界。

**当前阶段聚焦：**
- 中国股市（A股为主）
- 个人学习与产业理解工具
- 地理维度为核心探索方式
- 通过公开资料+AI辅助构建产业链知识

# 3. 产品定位（Positioning）

Industra 不是：

- 传统股票软件
- 行情工具
- 新闻资讯 App

它更像：

# Google Maps × Bloomberg × Palantir

构建的是：

# 下一代产业关系探索系统

（Next-generation Industry Intelligence Graph）

**差异化定位：**
- 不是看K线和涨跌，而是理解"为什么涨"
- 不是读新闻列表，而是看"新闻如何传导"
- 不是查公司资料，而是看"公司在产业中的位置"

# 8.2 边（Edge）

关系包括：

```text
属于（公司 → 产业）
供应（公司A → 公司B / 产品A → 产品B）
竞争（公司A ↔ 公司B）
投资（公司A → 公司B）
合作（公司A ↔ 公司B）
影响（事件 → 公司/产业）
替代（产品A ↔ 产品B）
位于（公司 → 地域）
包含（产业A → 产业B）
```

边属性：
- 关系类型
- 强度权重
- 时间范围（生效时间-失效时间）
- 来源
- 置信度

---

# 8.3 时间维度（Temporal Layer）

所有关系：

# 都必须带有时间属性

因为产业关系是动态变化的。

支持：
- 历史回溯
- 趋势分析
- 关系演变

# 11. 地理信息模块（Geo Module）

---

# 11.1 地理编码体系

```
国家 → 省 → 市 → 区/县 → 街道/园区
```

坐标系统：
- GCJ-02（高德/腾讯）
- BD-09（百度）
- WGS-84（国际标准）

---

# 11.2 企业地理位置获取

数据来源：
- 上市公司年报（注册地址/办公地址）
- 企查查/天眼查API
- 招股说明书
- 公司官网
- 产业园区公开信息

地理精度分级：
- Level 1: 省份（如"广东省"）
- Level 2: 城市（如"深圳市"）
- Level 3: 区县（如"南山区"）
- Level 4: 详细地址（如"科技园南区"）

---

# 11.3 空间分析功能

- **产业聚集度分析**：某产业在哪些区域最集中
- **跨区域对比**：不同省份/城市的产业结构差异
- **产业转移追踪**：企业搬迁、产能转移趋势
- **供应链地理分布**：上下游企业的空间关系
- **政策区域影响**：自贸区/高新区/经开区的产业分布

# 12.2 AI Prompt 模板

```
你是一个产业链分析专家。请分析【产业名称】的上下游关系。

# 12.3 数据质量保障

- **置信度评分**：每条关系标注可信程度
- **版本管理**：记录关系变更历史
- **用户反馈**：允许用户标记错误关系

3. 地图上显示AI基建相关企业分布（散点/热力）
4. 点击某个企业聚集区域（如"深圳"）
5. 显示该区域AI基建企业列表
6. 点击具体企业，查看详细信息和上下游关系
7. 点击上下游，继续探索整个产业链
```

# 13.2 交互设计原则

- **渐进式披露**：从宏观到微观，避免信息过载
- **空间记忆**：利用地图帮助用户建立空间认知
- **路径追踪**：显示用户探索路径，支持回溯
- **上下文保持**：切换视图时保持当前选择状态

# 19. 风险与挑战（Risks & Challenges）

---

# 数据质量

- **挑战**：公开数据准确性有限，产业链关系可能过时
- **应对**：多源交叉验证，标注置信度，用户反馈机制

---

# 数据获取

- **挑战**：部分数据源可能有反爬限制
- **应对**：优先使用API，合理控制爬取频率，手动补充

---

# 产业复杂度

- **挑战**：产业链关系错综复杂，边界模糊
- **应对**：聚焦核心关系，渐进式完善，接受不完美

---

# 技术复杂度

- **挑战**：大规模图数据渲染性能
- **应对**：分层加载，按需渲染，WebGL优化

---

# 维护成本

- **挑战**：产业关系持续变化，需要持续更新
- **应对**：AI辅助更新，社区贡献，版本管理
