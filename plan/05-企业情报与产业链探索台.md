# 企业情报与产业链探索台

## 1. 子产品定位

该子产品负责把用户从“看区域”带到“看企业”，并进一步进入“看上下游和竞争格局”的核心探索链路。

## 2. 目标

1. 建立统一企业详情页。
2. 打通企业与产业、区域、产品、上下游的浏览关系。
3. 提供适合学习者和投资者的探索型交互。

## 3. 核心职责

- 企业详情页
- 产业归属展示
- 上下游关系展示
- 竞争对手展示
- 相关产品与应用场景展示
- 探索路径回溯

## 4. 核心页面

### 4.1 企业详情页

- 基本信息
- 主营业务
- 所属产业
- 所在地区
- 财务摘要
- 相关新闻摘要

### 4.2 产业链关系面板

- 上游供应
- 下游应用/客户
- 同赛道竞争
- 替代关系

### 4.3 探索路径面板

- 用户当前浏览路径
- 上一步回退
- 收藏节点

## 5. 核心交互

```text
地图点击企业
-> 打开企业详情
-> 查看所属产业和上下游
-> 点击上游产品或公司
-> 切换到新的企业/产品节点
-> 继续沿产业链探索
```

## 6. 输入依赖

- 公司主数据
- 图谱关系数据
- 地理位置数据
- 事件系统输出的企业关联事件

## 7. 输出

- 企业详情聚合接口
- 企业上下游接口
- 企业竞争格局接口
- 用户探索轨迹上下文

## 8. MVP 范围

- 覆盖 AI 基建核心企业
- 支持企业基础画像
- 支持一跳上下游关系
- 支持简单竞争对手列表

## 9. 验收标准

- 用户从地图进入企业详情链路可用。
- 任一核心企业至少能看到产业归属和关键上下游。
- 企业页能和地图页、图谱页互相跳转。
- 浏览路径可以被记录和回溯。
