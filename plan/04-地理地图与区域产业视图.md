# 地理地图与区域产业视图

## 1. 子产品定位

地图模块负责**区域产业宏观视图**，不做单个公司的地理定位。核心价值是回答：

- "AI基建产业集中在哪些省份/城市？"
- "深圳的产业结构是怎样的？"
- "某省的龙头企业有哪些？"

**地图不是公司黄页，而是区域产业探测器。**

## 2. 设计原则（2026-05-17 修订）

### 为什么不在图上散点公司？

| 产业类型 | 地理相关性 | 举例 |
|----------|-----------|------|
| 能源/矿产 | **强** | 新疆光伏、内蒙风电——地理决定产业 |
| 制造/科技 | **弱** | GPU设计公司在哪对产业链理解无影响 |
| 软件/服务 | **几乎无关** | AI应用公司分布在全国各地 |

对于科技产业链（当前 MVP 范围），公司所在地对理解上下游关系几乎没有帮助。地理信息真正的价值是**区域产业聚集模式**（深圳=电子、上海=半导体、合肥=面板）。

### 正确的探索链路

```
地图（宏观）→ 区域产业面板 → 公司列表 → 公司详情+知识图谱
     ↑ 区域视角                          ↑ 关系视角
```

**地图管区域，图谱管关系。各司其职。**

## 3. 核心职责

- 全国省份产业密度热力图
- 省份聚合气泡（企业数量 + 产业分布概览）
- 点击省份 → 滑出区域产业面板（产业分布 + 代表企业列表）
- 城市级热力图（作为辅助图层）
- 产业筛选联动（侧边栏选产业 → 地图自动过滤）
- 省份筛选联动（地图选省份 → 企业列表自动过滤）

## 4. 不负责的内容

- 不在地图上散点展示单个公司
- 不在地图上做公司关系连线
- 不做省/市/区多级下钻（MVP阶段）
- 不负责公司详情展示（由企业探索台负责）
- 不负责知识图谱交互（由图谱视图负责）

## 6. 输入依赖

- 地域标准数据 + 企业坐标
- 产业分类结果（来自知识图谱引擎）
- 企业-产业-地域关联（来自数据底座）

## 7. 输出接口

- 区域产业分布查询 `GET /api/regions`
- 按省份/城市/产业筛选企业 `GET /api/companies?province=&industry_id=`
- 区域产业聚类统计

## 8. MVP 范围

- 全国地图 + 省份气泡
- 省份产业面板（产业分布 + Top 10 企业）
- 省份/产业双向筛选联动
- 城市热力图底层的静态展示

## 9. 验收标准

- [ ] 地图显示省份气泡，气泡大小按企业数缩放
- [ ] 悬停显示省份名+企业数+Top3产业，不阻挡点击
- [ ] 点击省份 → 右侧滑出 ProvincePanel
- [ ] ProvincePanel 显示产业分布条形图 + 代表企业列表
- [ ] 点击 ProvincePanel 中的企业 → 打开 CompanyDetail
- [ ] 侧边栏切换产业 → 地图气泡自动更新
- [ ] 点击地图空白 → 关闭 ProvincePanel
